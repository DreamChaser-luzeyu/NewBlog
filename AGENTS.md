# AGENTS

## 论文处理要求

当需要处理 `posts/论文阅读/` 目录下的 PDF 论文时，遵循以下规则：

1. 默认优先使用 `mineru_pdf_to_markdown.py` 工作流，通过 MinerU 将 PDF 转为 Markdown，并把 MinerU 返回的图片与资源保存到本地。
2. `mineru_pdf_to_markdown.py` 只负责生成抽取版 Markdown 文档，不在脚本内做全文翻译、导读生成或总结润色。
3. 若原论文是英文，需要在脚本生成的抽取版 Markdown 之外，再单独生成一份新的中文 Markdown 翻译文档。
4. 中文翻译文档开头必须明确声明：`文档由AI总结，可能有错误`。
5. 中文翻译文档开头需要加入一段面向外行的简明导读，说明：
   - 这篇文章研究的问题是什么；
   - 要解决哪些场景下的哪些问题；
   - 作者大致是怎么解决的。
6. 全文输出语言必须为中文。
7. 如果原论文是英文，需将全文翻译为中文，而不是只做摘要。
8. 正文应以“全文翻译 + 轻度润色”为原则：
   - 保留原文全部内容；
   - 不要擅自删减；
   - 不要改变作者原意；
   - 只做必要的中文表达润色，使其更通顺。
9. 不要生成过于简略的总结版文档；目标是尽量完整保留论文内容。
10. 论文封面、学位论文模板页、目录页等前置内容不应主导最终标题识别；标题应优先识别真实论文题目。
11. 导读生成优先基于摘要或正文前部关键内容，而不是仅基于封面信息。
12. 必须保留论文中的全部图表，并插入到 Markdown 文档中的对应位置。
13. 图表处理策略以稳定、完整为优先：
   - 不要使用 PDF 内嵌碎片图、logo 图、残缺图；
   - 优先使用 MinerU 抽取结果中的本地图片；
   - 如果 Markdown 中引用的是 MinerU 服务器上的远端图片，必须下载到本地后再改写为本地路径；
   - 图片资源路径最终必须指向 `posts/论文阅读/mineru_assets/<论文名>/` 下的本地文件。
14. MinerU 标准 API 适合正式处理：支持更大文件、更多页数、zip 结果与资源文件；Agent 轻量 API 适合较小文件的快速测试。
15. 对抽取版 Markdown 和中文翻译文档中的 LaTeX 公式、百分比、倍率、单位、编程语言名等表达式进行检查；若存在明显的抽取错误、断裂、误加数学定界符或错误转义，必须结合上下文修正。
16. 处理前应先 `source secret_env.sh` 或等效方式加载 MinerU 所需的私有环境变量。

## 执行偏好

1. 对论文内容的处理优先遵循“完整、忠实、可读”，而不是“简短、概括”。
2. 对图表内容的处理优先遵循“完整展示”，而不是“只截取看起来像图表的一小块区域”。
3. 当 PDF 内嵌图片资源质量不可靠时，优先使用整页转图 + 裁剪回退策略。
4. 当翻译模型可用时，优先用模型生成外行导读；若导读生成失败，则退回到基于摘要或正文前部内容的规则化导读。
5. 处理前应先 `source secret_env.sh` 或等效方式加载 MinerU 与翻译模型所需的私有环境变量。
